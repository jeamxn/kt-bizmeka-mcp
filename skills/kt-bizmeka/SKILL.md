# KT 비즈메카 EZ 자동화 — 전체 안내 (man)

KT 비즈메카 EZ 포털과 웹메일을 파이썬 패키지 `kt_bizmeka_mcp`의 함수를 직접 호출해 다룬다.
이 스킬은 **작업 전 먼저 읽는 목차/지도**다. 실제 절차는 작업별 세부 스킬에 있다.

## 작업별 세부 스킬

| 하려는 작업 | 사용할 스킬 |
|---|---|
| 로그인 (아이디/비번 + SMS 2차인증) | **kt-bizmeka-login** |
| 메일함 조회 · 메일 내용 읽기 · 첨부 다운로드 | **kt-bizmeka-mail-read** |
| 메일 발송 · 답장 · 수신자 검증 | **kt-bizmeka-mail-send** |
| 보낸 메일 수신확인 · 발송취소 | **kt-bizmeka-mail-receipt** |

거의 모든 작업은 **로그인이 선행**되어야 한다. 먼저 kt-bizmeka-login으로 로그인한 뒤, 같은 프로세스 안에서 메일 작업 스킬의 코드를 이어붙여 실행한다.

## 실행 환경

이 스킬이 설치된 디렉토리(패키지 루트)에서 `uv`로 실행한다. 의존성은 `uv run`이 자동 동기화한다.

```bash
cd <패키지 루트>        # 이 스킬들이 들어있는 repo / 플러그인 캐시 경로
uv run python <스크립트>
```

핵심 객체는 `kt_bizmeka_mcp.client.BizmekaClient` 하나다. 이 인스턴스가 쿠키/세션/CSRF를 전부 들고 있으므로, **로그인부터 메일 작업까지 같은 프로세스·같은 client 객체로 이어가야 한다.**

## 공통 핵심 제약 (모든 작업에 적용)

- **SMS 2차 인증이 필수**다. 사람이 휴대폰으로 받은 인증번호를 입력해야 하므로 로그인은 한 프로세스 안에서 "SMS 발송 → 사용자 입력 대기 → 인증" 순으로 진행한다. (kt-bizmeka-login의 러너 패턴)
- 인증번호 유효시간은 약 3분. 만료되면 처음부터 다시.
- **client 객체를 프로세스 밖으로 직렬화해 재사용하지 말 것.** httpx 세션이 끊겨 인증이 깨진다. 부득이하면 `dump_cookies()`/`load_cookies()`로 쿠키만 넘기되, 로그인 직후 한 프로세스에서 끝내는 것이 안전하다.
- **메일 발송/답장/발송취소는 실제 부작용**이 있다. 호출 전 수신자·제목·본문을 사용자에게 반드시 확인받는다.
- 함수 실패 시 `kt_bizmeka_mcp.client.BizmekaError`를 던진다. 메시지(세션 만료/인증번호 오류 등)를 그대로 사용자에게 전달한다.
- 메일 함수는 첫 호출 시 `enter_webmail()`(웹메일 SSO + `_csrf` 확보)을 자동 수행하므로 직접 부를 필요 없다.

## 패키지 구조 요약

- `kt_bizmeka_mcp.client.BizmekaClient(username, password)` — 로그인 세션 객체
- `kt_bizmeka_mcp.mail` — 웹메일 작업 함수 모음 (첫 인자는 항상 위 client)

각 함수의 정확한 시그니처와 예제는 해당 작업 스킬에서 확인한다.
